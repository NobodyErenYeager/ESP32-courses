This lecture has not been recorded yet, however, I would like to inform everyone that when using ESP-IDF version 5 and above, you will need to increase your partition sizes for the ota_0 and ota_1 slots. With IDF version 5 and above, the application size has increased and to allow for more space, we are removing the "factory" app slot and using only ota_0 and ota_1 for the running application / firmware updates and swapping between the two.

Prior to ESP-IDF version 5, we did not have to increase the partition sizes until the AWS lessons, which is why updating the partition table is demonstrated there. Until I am able to update this section, I'm suggesting that you use the partition table from the AWS lessons (attached to this lesson as well).

Take this .csv file and paste it into your ESP-IDF partition_table folder path. For example, my path is; C:\Espressif\frameworks\esp-idf-v5.0.1\components\partition_table. Be sure to clean and rebuild after you've added the updated file.

And, as a friendly reminder, please be sure that you've updated the SDK Configuration as we did in Lesson 7 (Eclipse Project Template Setup) --> we need a flash size of at least 4 MB, partition table set to use Factory app, two OTA definitions and Max HTTP Request Header and URI Lengths to 1024.
